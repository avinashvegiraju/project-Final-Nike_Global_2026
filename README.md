# Nike Global Catalogue 2026 Final Project

## Overview
This project analyzes Nike's global product catalog snapshot from **March 19, 2026** using a scalable workflow built around **DuckDB**, **Pandas**, **SQL**, and a **Streamlit + Plotly** dashboard.

The project is intentionally framed as a **cross-market assortment, availability, and same-currency price-positioning study**. It does **not** claim time-series trends because the source data is a single snapshot.

## Business Questions
- Which markets have the deepest assortment breadth?
- Which markets show the strongest stock exposure and size availability?
- Where is discounting most prevalent?
- For products sold in multiple countries that share a currency, where are the biggest price gaps?

## Project Deliverables
- Reproducible large-file data build in DuckDB
- Cleaned and aggregated analytical tables
- SQL analysis layer for profiling and benchmarking
- Statistical analysis outputs and insight summary
- Multi-page Streamlit dashboard reading only exported aggregated data

## Build The Analytical Assets
```bash
python src/build_assets.py
```

This step:
- loads either `../Nike_global/Global_Nike.csv` or the `../Nike_global/Nike_*.csv` country files into DuckDB
- validates schema and snapshot constraints
- cleans booleans and pricing logic
- creates product-market, market, and pricing benchmark tables
- exports dashboard-ready Parquet files into `exports/`

Optional dataset overrides:
```bash
NIKE_RAW_DATA_DIR=/absolute/path/to/Nike_global python src/build_assets.py
NIKE_RAW_CSV=/absolute/path/to/Global_Nike.csv python src/build_assets.py
NIKE_RAW_GLOB='/absolute/path/to/Nike_global/Nike_*.csv' python src/build_assets.py
```

## Run The Statistical Analysis And Insight Report
```bash
python src/run_analysis.py
```

This step:
- runs the required statistical tests at the product-market level
- exports contingency tables and audit tables to `exports/`
- generates `docs/INSIGHT_SUMMARY.md`

## Key Data Rules
- Price comparisons are **same-currency only**.
- Cross-country benchmark grain is `country_code + currency`.
- `AU`, `EG`, `IN`, and `NZ` are treated as **low-coverage countries** and excluded from peer ranking visuals by default.
- Mixed-currency submarkets such as `BG|BGN` are preserved for pricing logic and flagged when sparse.

## Project Caveats
- This is a **catalog snapshot**, not sales or inventory history.
- `sale_price_local` is sparse, so discount logic is normalized from both `sale_price_local` and `discount_pct`.
- Some columns in the raw data are effectively unusable for analysis (`employee_price`, `gtin`, `size_count`, `available_size_count`) and are excluded from the cleaned model.
